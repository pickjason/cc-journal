# 2026-06-11 · 开源准备(P0 + P1 部分)

> 目标:陌生用户路径压缩为「看到截图 → `npx cc-journal serve` → 30 秒看到自己的热力图」。
> 完成 P0 全部(npm 化 / 双语 README + 截图 / LICENSE)+ P1 的 CLI 双语与跨平台修复。

## 时间线

1. **包名查重**:`claude-journal` 已被占(npm 上 v1.4.1),改用 **`cc-journal`**(已确认未占用)。bin 提供 `cc-journal` 与 `journal` 双名。
2. **npm 化三处关键改造**:
   - `--data-dir` 默认值从 `<repo>/data` 改为 **`~/.claude-journal/`**(npx/全局安装时包目录不可写);本机旧数据已迁移(`cache.json` + 日报缓存,旧 `2026-06-11-llm.md` 更名为 `-llm.zh.md`)。
   - ECharts 从 jsdelivr CDN 改为**本地 vendor**:`scripts/copy-vendor.mjs` 在 build 时从 node_modules 拷到 `web/vendor/`(gitignore,echarts 进 devDependencies);境内/离线均可用。
   - `package.json`:`files: [dist, web, README.zh-CN.md]`、`prepare` 钩子自动构建、repository/keywords/license 元数据齐全。
3. **CLI 双语**(`src/i18n.ts`):`detectLang()` 优先级 `--lang` > `JOURNAL_LANG` > `LC_ALL/LC_MESSAGES/LANG` > Intl locale,默认 en。`CLI` 字典覆盖 stats/serve/refresh/summary 全部输出;`SUM` 字典覆盖规则日报与 LLM prompt(英文版要求英文日报)。`/api/summary` 增加 `lang` 参数,Dashboard 按当前界面语言请求,LLM 缓存按语言分文件 `<date>-llm.<lang>.md`。
4. **跨平台**:打开浏览器 darwin `open` / win32 `cmd /c start` / 其他 `xdg-open`;`readdir` 的 `parentPath`(Node ≥20.12)加 `path` 字段兜底,`engines: >=20`。
5. **README 重写**:英文 `README.md` 为主(首屏 npx 一条命令 + Dashboard 截图 + 隐私声明),`README.zh-CN.md` 内容对齐互链;统计口径、30 天清理与历史积累、FAQ 全覆盖。截图用 Chrome headless 对 `?lang=en` 页面渲染(`docs/screenshot.png`),README 引用 raw.githubusercontent URL(GitHub 与 npm 页面都能显示)。
6. **LICENSE**:MIT,2026 pickjason。

## 验证

- `tsc` 通过;`stats --lang en` / `--lang zh` 输出对齐正确;英文规则日报标题/概览为英文(指令原文保留原语言)。
- 迁移后的 `~/.claude-journal/cache.json` 直接被新默认路径读到,数据无损。
- Dashboard 用本地 vendor 的 echarts 正常渲染(已断开 CDN 依赖)。
