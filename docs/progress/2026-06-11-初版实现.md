# 2026-06-11 · claude-journal v0.1 初版实现(从零到全功能)

> 一天内从空仓库到可用工具:CLI 三命令(stats/serve/summary)+ Dashboard(热力图/趋势/时段/项目/模型/当日明细)+ 中英文切换 + 页面日报(LLM 版带缓存)。
> 全部经真实数据与浏览器验证;尚未做首次 git 提交。

## 时间线

1. **方案确认**:产品形态(CLI + 本地网页 Dashboard)、技术栈(Node 20 + TypeScript,唯一运行时依赖 commander)、日报来源(规则提取 + 可选 LLM 浓缩)三项经用户选定后开工。LLM 明确走本机 `claude -p`(订阅,不配 API key),默认 haiku。
2. **数据源勘探**:`~/.claude/projects/` 25 个项目 / 211 个 jsonl / 759MB。验证字段:assistant 行带 `message.usage`(`input_tokens`/`output_tokens`/`cache_creation_input_tokens`/`cache_read_input_tokens`)、`message.model`、`requestId`;通用字段 `timestamp`(UTC)/`sessionId`/`cwd`/`gitBranch`/`isSidechain`。
3. **骨架落地**:`scanner → parser → cache → aggregate` 数据流 + `cli/server/summary` 三出口 + `web/`(原生 JS + ECharts CDN,深色 GitHub 风)。
4. **🔍 两个数据问题(本日关键发现)**:
   - **子代理转录不在主文件里**:新版 Claude Code 把 subagent 拆到 `<session>/subagents/agent-*.jsonl`(126 个文件,占总文件数过半),一级扫描全漏。改递归扫描,这类文件 `forceSidechain`:用量计入、不算会话。
   - **源数据只回溯 30 天**:Claude Code 默认 `cleanupPeriodDays=30` 自动清理,所以仅有 2026-05-08 起的数据。对策:`cache.ts` 刻意**不删失踪源文件的缓存条目**,journal 从首次运行起长期积累历史(由此 `data/cache.json` 成为唯一历史,列入红线)。
5. **核心验证(全过)**:
   - 性能:首次全量解析 ~3s,增量刷新 0.4s(按 size+mtime 判断)。
   - 数字:近 30 天 78 会话 / 1016 指令 / 输出 15.9M tokens;cache 读写 4.7B(与 input/output 约 300:1,印证四项分开统计的必要性)。
   - 日报:规则版能准确还原当天各项目工作(含本会话自身);LLM 版(haiku)约 15–20s。
6. **中英文切换**:文案抽到 `web/app.js` 顶部 `I18N` 字典(含 ECharts 序列名/日历星期月份/tooltip),`localStorage` 记忆 + `?lang=zh|en` 参数 + 默认跟随浏览器语言。中英两版均经页面文本验证。
7. **页面日报**:`/api/summary`——规则版即时现生成;LLM 版缓存 `data/summaries/<date>-llm.md`(实测首次生成 14.8s,缓存命中 0.18s),`force=1` 重新生成,服务端 inflight Map 防并发重复烧用量。前端自写 `mdToHtml()` 渲染(内容先转义),按钮/状态文案进 i18n。
8. **防自指污染**:LLM 日报 prompt 以 `[journal-summary]` 开头,parser 检测到即整会话 `excluded`——验证过生成日报后会话数不增。

## 教训

- **Chrome 截图通道故障 ≠ 页面故障**:验证中 `screenshot` 连续超时,先怀疑前端死循环,实际 `get_page_text` 正常返回(渲染进程活着),curl 服务端也全通——是浏览器扩展截图功能临时故障,稍后自愈。判障要多通道交叉,别盯着单一信号重试。
- **同一响应多行重复 usage**:jsonl 里一次 API 响应的每个 content block 各占一行且都带完整 usage,不按 `message.id + requestId` 去重会数倍虚高。
- zsh 里 curl 带 `?` 的 URL 必须加引号(`no matches found`)。
